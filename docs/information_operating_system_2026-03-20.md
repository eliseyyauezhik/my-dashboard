## Правило обработки новой информации
- **1. Захват** — Каждый новый источник или элемент получает путь, дату, тип, жизненную зону и признак дубликата.
- **2. Сжатие** — Извлекается суперкороткое резюме: суть, сигнал, теги, возможное применение, следующий шаг.
- **3. Классификация** — Материал раскладывается по осям: жизненная зона (я, семья, работа) и тема (AI, образование, дом, бизнес и т.д.).
- **4. Привязка** — Перед созданием новой сущности проверяется связь с существующим проектом, skill, workflow, идеей или заметкой.
- **5. Решение** — Элемент должен завершиться одним исходом: обновление проекта, новая идея, новая задача, skill-candidate, reference-note или архив.
- **6. Ревизия** — На обзор поднимаются только новые дельты и топ-сигналы; весь архив не перечитывается.

## Маршрутизация входящего
- **Есть прямая связь с активным проектом** → Обновление карточки проекта. Добавить краткое резюме, теги, источник и следующий шаг в существующий проект.
- **Появился повторяющийся паттерн действий для агента** → Кандидат в новый skill. Оформить навык с применением, входами, выходами и правилами использования в CLI/IDE.
- **Есть сильный сигнал, но проекта еще нет** → Новая проектная гипотеза. Создать короткую проектную карточку: цель, ценность, ближайший эксперимент, связи.
- **Материал полезен, но не тянет на проект** → Reference-note в базе знаний. Сохранить компактную заметку с сутью, тегами, принципами применения и ссылкой на источник.
- **Материал не несет действия или связи** → Архив. Оставить только путь и минимальный индекс без перегруза основного контура.

## Рекомендации
- Считать Telegram, YouTube и другие потоки входным слоем, а не местом долгого хранения решений.
- Создавать новую сущность только после проверки связи с уже существующим проектом или skill.
- Держать суперсжатый формат карточек: суть, сигнал, теги, применение, next step, связи.
- Навыки для агентов создавать по повторяющимся паттернам, а не по единичным интересным находкам.
- Раз в неделю смотреть только топ-дельты и источники с наибольшим рангом, а не весь архив.
- Держать отдельный слой triage для входящего, а не складывать сырые данные сразу в проекты.
- Считать Saved Messages и экспортированные каналы разными типами источников: личный inbox и тематические потоки.
- Каждую неделю пересматривать только топ-источники по рангу и новые дельты, чтобы не тонуть в историческом хвосте.
