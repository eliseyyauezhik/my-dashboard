# Размещение дашборда: GitHub сейчас, РФ потом

## 0) Что именно хостим

- Дашборд сейчас статический: `index.html`, `app.js`, `style.css`, `mindmap.html`, `mindmap.js`, `data/*.json`.
- Запись идей в `data/idea_inbox.json` возможна только через локальный API (`scripts/dashboard/dashboard_server.py`). На чистом статическом хостинге POST не будет работать.

## 1) Быстрый старт: GitHub Pages (статический)

- В репозиторий кладется статическая сборка (файлы из корня + папка `data/`).
- Включается GitHub Pages для ветки/папки (обычно `main` + `/`).
- Обновление данных происходит вручную: сначала прогон `.\scripts\dashboard\sync_workspace_data.ps1`, потом коммит/пуш.

Примечание: внимательно проверьте, что в `data/*.json` нет приватного (телефоны, токены, внутренние ссылки).

## 2) Переезд на стабильный российский хостинг (2 варианта)

### Вариант A: только статика (проще, дешевле, быстрее)

- Объектное хранилище с static website + CDN:
- Дашборд отдается как набор файлов, обновление через загрузку новой версии (или CI/CD).
- Идеи сохраняются локально (через `dashboard_server.py`) или вручную в `data/idea_inbox.json`.

### Вариант B: статика + маленький backend (если нужен POST /api/idea-inbox)

- Статика на CDN/объектном хранилище.
- Backend на самом дешевом VPS в РФ (Nginx + Python), который:
  - отдает `/api/idea-inbox` и принимает `/api/idea-inbox/ideas`;
  - пишет в файл `data/idea_inbox.json` (или в маленькую БД).
- Доступ закрывается Basic Auth или IP allowlist.

## 3) Практичное правило

- Если дашбордом пользуетесь только вы: держите прод как статический, а запись идей делайте локально.
- Если нужен публичный доступ и запись идей: делайте отдельный backend, не пытайтесь “прикрутить” запись к GitHub Pages.
