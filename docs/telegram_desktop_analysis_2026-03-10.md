## Предложения по навыкам
- **telegram-export-intelligence** — Навык для разбора экспортов Telegram Desktop в компактные карточки знаний, связи и рекомендации.
- **saved-messages-triage** — Навык для превращения Saved Messages в решения: task, note, project, archive.
- **channel-idea-distiller** — Навык для сжатия AI-каналов и подборок ботов в применимые паттерны и идеи для агентов.
- **family-ops-briefing** — Навык для семейных чатов: резюме, решения, события, покупки и контрольные списки.
- **education-grant-radar** — Навык для систематизации школьных/грантовых идей и связывания их с текущими образовательными проектами.
- **project-context-allocator** — Навык для автоматического распределения входящего по уже существующим проектам вместо создания дублей.

## Протокол обработки входящего
- **1. Захват** — Каждый новый источник получает карточку с путём, типом, датой, объёмом и признаком дубликата.
- **2. Сжатие** — Поток сразу переводится в короткое резюме: суть, сигналы, теги, возможное применение, а не в длинный пересказ.
- **3. Классификация** — Материал распределяется по двум осям: жизненная зона (я, семья, работа) и тема (AI, бизнес, образование, дом и т.д.).
- **4. Привязка** — Перед созданием новых сущностей проверяется связь с уже существующими проектами, workflows, навыками и областями интереса.
- **5. Решение** — Каждый элемент должен закончиться одним из статусов: задача, идея проекта, обновление skill, reference-note или архив.
- **6. Ревизия** — Раз в неделю смотрятся только топ-источники и новые дельты, а не весь архив целиком.

## Рекомендации
- Держать отдельный слой triage для входящего, а не складывать сырые данные сразу в проекты.
- Считать Saved Messages и экспортированные каналы разными типами источников: личный inbox и тематические потоки.
- Каждую неделю пересматривать только топ-источники по рангу и новые дельты, чтобы не тонуть в историческом хвосте.
- Новые skills предлагать на основании повторяющихся паттернов, а не единичных интересных сообщений.

## Риски
- В `telegram_inbox.py` лежат чувствительные токены и ключи в открытом виде; это стоит вынести в переменные окружения.
- Часть экспортов продублирована и есть повреждённые JSON; для истории это нормально, но для автоматизации нужен явный слой дедупликации.
- Без маршрутизации по жизненным зонам идеи из семьи, работы и личных интересов конкурируют в одной общей очереди.
