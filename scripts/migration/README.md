# Project Migration To Yandex Disk

Набор для безопасного копирования проектов в одну папку Яндекс.Диска без удаления исходников.

## Что делает

- читает список проектов из `projects_manifest.json`
- копирует их в `D:\ЯндексДиск\Yandex.Disk\ПРОЕКТЫ\AI_Workspace\project_archive`
- по умолчанию работает в `dry-run`
- пишет CSV-отчёт по каждой операции
- по умолчанию исключает тяжёлые служебные каталоги вроде `.git`, `node_modules`, `.venv`, `__pycache__`

## Почему это безопаснее, чем move

- исходные проекты остаются на месте
- можно сначала проверить план и только потом запускать реальное копирование
- отчёт показывает, что скопировано, что пропущено и что отсутствует
- исключение кэшей и зависимостей уменьшает объём и ускоряет синхронизацию Яндекс.Диска

## Рекомендуемый сценарий

1. Сначала dry-run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\migration\copy_projects_to_yadisk.ps1
```

2. Проверить JSON в консоли и CSV-отчёт в папке `_migration_reports`.

3. Если всё нормально, выполнить реальное копирование:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\migration\copy_projects_to_yadisk.ps1 -Execute
```

## Полезные флаги

- `-Execute`:
  включает реальное копирование

- `-IncludeDisabled`:
  включает отключённые в манифесте элементы, например `C:\Users\Admin\.copaw`

- `-IncludeHeavyArtifacts`:
  копирует также `.git`, `node_modules`, `.venv` и другие тяжёлые каталоги

- `-DestinationRoot 'D:\...\ДругаяПапка'`:
  позволяет переопределить корневую папку назначения

- `-ReportPath 'D:\...\report.csv'`:
  позволяет задать путь отчёта вручную

## Когда лучше другой вариант

Если цель не архивирование, а именно ежедневная работа из одного места, лучше не копировать всё подряд, а сделать двухслойную схему:

- слой 1: `AI_Workspace\project_archive` для облачных копий исходников
- слой 2: `AI_Workspace\indexes` для CSV/MD-индексов, чтобы быстро искать проект, чат и workflow

Это надёжнее, чем физически переносить рабочие каталоги, потому что не ломает текущие абсолютные пути, скрипты и деплой-настройки.
